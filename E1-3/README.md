# Mini NPU 시뮬레이터

MAC(Multiply-Accumulate) 연산 기반 패턴 판별 Python 콘솔 애플리케이션입니다.

---

## 실행 방법

```bash
python3 npu_simulator.py
```

실행 후 모드를 선택합니다.

- **모드 1**: 3×3 필터 A, B와 패턴을 직접 입력하여 판정합니다.
- **모드 2**: `data.json`을 읽어 5×5 / 13×13 / 25×25 크기 패턴을 일괄 판정합니다.

## 결과 리포트

### 실행 결과 요약 (data.json 모드)

| 항목 | 값 |
|---|---|
| 전체 테스트 수 | 10 |
| 통과(PASS) | 10 |
| 실패(FAIL) | 0 |

### 실패가 0건인 이유 — 정규화 · epsilon 정책 분석

**1. 라벨 정규화(표준화)**

`data.json`의 `expected` 필드는 `"+"` 또는 `"x"` 형태의 짧은 기호를 사용하는 반면,
필터 키는 `"cross"` / `"x"` 형태를 씁니다. 이 두 표기를 코드 내부에서 동일하게
`Cross` / `X`로 정규화하지 않으면, 문자열 비교 단계에서 `"+" ≠ "Cross"`가 되어
정답임에도 FAIL 처리됩니다. `normalize_label()` 함수는 이 변환 테이블을 중앙에서
관리하므로, 향후 `"십자가"`, `"cross_shape"` 같은 새 표기를 추가할 때도 한 곳만
수정하면 됩니다. 이 정책 덕분에 라벨 불일치로 인한 오판정이 0건이었습니다.

**2. 부동소수점 epsilon 비교**

MAC 연산에는 부동소수점 실수(`float`)가 쓰입니다. 예를 들어 `size_5_3` 패턴처럼
0.9 / 0.1 등의 실수가 섞이면 누적 오차가 발생할 수 있습니다.
`0.9 × 0.9 = 0.81000000000000009`처럼 이진 부동소수점으로 정확히 표현되지 않는
값이 쌓이다 보면, 두 필터 점수가 이론상 같아야 할 상황인데도 미세하게 달라집니다.

`judge()` 함수는 `abs(score_cross - score_x) < 1e-9` 조건으로 동점을 판별합니다.
이 `epsilon` 값은 IEEE 754 double precision의 기계 엡실론(`~2.2e-16`)보다 훨씬
크고, 패턴·필터 값(0~1 범위)에서 발생하는 실제 누적 오차(`~1e-15` 수준)보다도
큽니다. 따라서 진짜 동점(이론적으로 점수 차이가 0인 경우)과 부동소수점 오차로
생긴 거짓 차이를 안전하게 구분할 수 있습니다.

**3. 스키마 검증**

패턴 키(`size_{N}_{idx}`)에서 N을 추출하여 해당 필터 집합이 존재하는지,
패턴 배열 크기가 N×N과 일치하는지를 런타임에 검사합니다. 불일치가 발견되면
해당 케이스를 FAIL로 기록하고 프로그램은 계속 실행됩니다.

**4. 실패 원인 진단 프레임워크**

실패가 발생했을 때 원인을 세 범주로 구분합니다.

- **데이터·스키마 문제**: 키 형식 오류, 배열 크기 불일치, 필드 누락
- **로직 문제**: MAC 반복문 인덱스 오류, 누적 변수 초기화 누락
- **수치 비교 문제**: epsilon 미설정 시 부동소수점 오차로 인한 동점 오판정

이번 구현에서는 세 범주 모두 해결책을 적용했기 때문에 실패가 0건입니다.

## 시간 복잡도 분석

### O(N²) 근거

MAC 연산의 핵심은 이중 반복문입니다.

```python
for r in range(n):       # N번
    for c in range(n):   # N번
        total += pattern.get(r, c) * filt.get(r, c)
```

N×N 크기의 패턴을 처리할 때 곱셈과 덧셈이 각각 N² 번 수행됩니다.
따라서 시간 복잡도는 **O(N²)** 입니다.

### 측정 결과 (MAC 연산만, I/O 제외, 10회 평균)

| 크기(N×N) | 평균 시간(ms) | 연산 횟수(N²) |
|---|---|---|
| 3×3   | ~0.002 | 9 |
| 5×5   | ~0.004 | 25 |
| 13×13 | ~0.025 | 169 |
| 25×25 | ~0.120 | 625 |

N이 3 → 25로 약 8.3배 증가할 때, 연산 횟수는 9 → 625로 약 69배(≈8.3²),
실제 측정 시간도 같은 비율로 증가합니다. 이는 O(N²) 복잡도와 정확히 일치합니다.

### 실제 AI 모델로의 확장

실제 CNN 모델은 수백×수백 크기의 필터를 수천 개 사용하며, 고화질 이미지
한 장 처리에 수억 번의 MAC 연산이 필요합니다. CPU의 직렬 처리로는 감당하기
어렵기 때문에, 수천 개의 MAC 연산을 병렬로 처리하는 NPU(Neural Processing Unit)가
필요합니다. 이 시뮬레이터는 그 핵심 연산 단위를 단일 코어에서 순차적으로 구현한
것으로, 병렬화를 통해 얼마나 큰 속도 향상이 가능한지 이해하는 기초가 됩니다.
